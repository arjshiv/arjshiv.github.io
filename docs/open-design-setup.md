# Open Design Setup

Reference source: https://github.com/nexu-io/open-design

Local setup for this pass:

- Cloned to `/tmp/open-design` as a reference checkout.
- Used the craft rules in `craft/anti-ai-slop.md`, `craft/color.md`, and `craft/typography.md`.
- Used the `critique` and `tweaks` skill patterns as product inspiration, not as vendored site code.

Rules applied to this site:

- Keep one clear design direction: bright cool field-passport, not generic SaaS or dark terminal.
- Use accent sparingly. Most readable text stays ink on paper, cyan, aqua, or mint.
- Treat all-caps labels as labels only, with tracked mono type.
- Make the page useful, not just decorative: browser tools should answer, critique, route, or clarify.
- Review the work against Open Design's five dimensions: philosophy, hierarchy, detail, function, and innovation.

The full Open Design repository is intentionally not committed here. This site should carry the adapted design decisions, not another dependency tree.
